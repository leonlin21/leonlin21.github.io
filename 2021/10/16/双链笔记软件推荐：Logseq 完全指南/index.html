双链笔记软件推荐：Logseq 完全指南

开篇

在这接近 15 年的网上冲浪生涯中，我确信我锤炼得最有心得的能力有两个，第一个是在群友发出色图挑战后，在 3 分钟内帮他找到图片来源并给出磁力下载链接。第二个便是捣鼓各种在别人看起来相当无聊的笔记软件并能从中获得乐趣。

这两者的共同点在于，这个过程总是让我能开拓眼界和三观，并达到某种贤者状态。每当我被新的知识荡涤心灵后总会惊呼：这东西原来还能这么玩？属于是吾生也有涯，而姿势也无涯了属于是。嗯，我指的是各种各样的笔记软件。

后来我遇到了 Logseq，它确实改变了我记笔记的方式和习惯，在这半年内我用它写下了超过 30 万的文字，比过去 5 年还多。就像在 P 站翻了九九八十一页后，突然间停下了寻觅的双眼，尘埃落定，从此告别用过的软件比写下的笔记还多的局面。

所以 Logseq 是什么，它能做到什么，为什么是它而不是别家的笔记？看完这篇文章，相信你对它就能有个七八成的认识。但我并不想神话双向链接有多么好用，我只是想简单呈现一下双向链接都能做到什么，以及它带给我的启发是什么，以至于让我选择了它。

须知任何笔记都有缺点， Logseq 的很多功能在别的软件或许更好，没问题，大家各取所需。

Logseq 是什么

这是一款由国人开发的支持双向链接的高质量大纲笔记软件，双向链接这个概念先不展开，先介绍几个感知强烈的优点。

首先，它现在 beta 测试阶段的主要功能包括现在及以后都将永久免费。其次，它秉承隐私至上的理念，你可以完全离线使用它。所有文档将以 Markdown 形式存储在本地，Windows 自带的 txt 编辑器，或者 Typora、Obsidian、VScode 随便哪款都可以打开它。即便将来你放弃使用，文档的迁移也很简单，你的知识将永远只属于你。

整个软件基于六大系统构筑而成：日志系统、双向链接系统、GTD 系统、PDF 标注系统、闪卡系统、插件系统，每个系统合理交织，我将逐一介绍它的使用方法。

基本界面

安装并打开软件后，你可以通过点击【Open a local directory】来定位你的【笔记存储位置】到本地任意一个文件夹中。如果是将文件夹放在坚果云或者群晖等同步盘中，就能够实现多端同步。

如你所见，Logseq 是一款大纲结构的笔记软件，你可以暂时先把它当成幕布（或 workflowy）来用，刚好 Logseq 同样也能生成思维导图。但看下去你就会明白，它可能会比传统的大纲笔记强不少。

对笔记软件有需求的人大概有这几类：要做学科笔记的学牲，心血来潮想复盘工作的社畜，想做读书笔记的学者，或者是想随手记录灵感和心情的一般人。接下来我将从这些人的视角，模拟在各种需求下，我们可以怎么样使用 Logseq，然后步步进阶。

用法一：全功能型日记本

Logseq 的最基础面板是【日志】，只要你打开它，它每天都会自动生成一页空白的日志（daily note），并以日期作为它的标题。在这里每一天都是新的开始，但今天你所记录的一切，在未来将会以一种奇妙的方式于你眼前再现。

  • 支持文档历史版本控制
  • 支持 PDF 阅读与标注
  • 支持 Markdown 输入和简单表格
  • 支持 LaTeX 和代码输入
  • 可插入图片和本地视频
  • 可嵌入 HTML 和在线视频窗口（哔哩哔哩、YouTube，其中油管支持时间戳）
  • 可嵌入能自由绘画的画板
  • 纯本地、支持加密、免费，好用（PC、Mac、Linux）
  • 移动端开发中

你可以暂时把这个每天自动生成的日志页面当成一个 inbox，不论是灵感、吐槽、备忘，任何你想记录的东西都可以暂时放在这里。

通过插件系统，它也能实现类似 flomo 这种的类 Github 的笔记热力图。当天写的日记越长，它的颜色就深。深色方块越多，就代表那段时间你越认真努力。

点击热力图上的小方块，可以快速筛选写过日志的日期，通过点击能直接跳转那一天。

用法二：GTD 任务管理系统

Logseq 内置了一套虽然基础但基本够用的 GTD 系统。假设我是一个钱少事多离家远的社畜，我可以怎么使用 Logseq？首先，每天早上我会在日志中花 5 分钟来梳理一下今天的工作。

但只是简单的清单罗列，好像还不算任务管理系统，所以你可以通过斜杠 / 来添加 todo 标记，并在完成每条任务后打钩。

如果这个任务是未来某天需要完成的，你可以通过 / 唤出 date picker 命令来添加时间，例如 10 月 31 号。

当 31 号那天到来，这个任务就会再次出现在当天的日志中，起到提醒的效果。

如果使用 doing 或者 now 命令，它会更显眼地出现在每天日志的下方（如红框所示），以防当天记录的东西过多，或者到了第二天生成了新的日志后被忽略。直到你将它完成为止，它才会消失，算是一种强提醒。

你还可以使用 Query 命令，来筛选、汇总你分散在任意页面的所有任务，这样一来是不是稍微有点“任务管理”那味儿了。

此处只展示了最基础的 Query 命令，感兴趣的话可再自行探索进阶用法。

用法三：记录工作日报和周报

Page 的概念

在实操工作日报之间，我还需要先引入一个名为 Page 的概念。

你可以在【任意位置】使用方括号[[ ]] 将某个关键词框柱，Logseq 就会新建一个以这个关键词命名的页面，点击就可以进入这个 Page 中，这里类似 Notion 的/ page 功能。另外，在中英文输入法下，双击[ 都能快速补全[[]]。

日报实操

我每天都会在 daily note 中记录我的工作日报，点击【工作日报】关键词，就能进入这个页面中，并看到所有带这个关键词的日志，都被按时间顺序汇总到了这个页面下方的【Linked Reference】中。

下面图示中的每个长长的黑色箭头，都代表一个【点击关键词并跳转】的动作。

因为这里的【工作复盘】也是一个关键词，所以它也是可点击的。进入【工作复盘】这个 Page 后，你可以看到每天的复盘都将按照时间顺序汇总好。

这个动作能让你将所有的【工作复盘】从所有的【工作日报】中筛选出来，让你能更聚焦地只看这一项，方便你按照时间线回顾这一段时间以来的经验心得。

如果你在每天的【工作日报】后面附上当周的 tag，例如 #week18 ，那么当一周结束后，只要你点击这个 tag，就能汇总当周的所有日报，以便你能更好地写周报。（实际上这里是汇总了所有打上了 #week18 的内容）

但这种用法，和打标签有什么区别？印象笔记可以打标签，Bear 也可以打标签，如果用 Notion 记日报，还有多种视图可以切换，用滴答清单记日报，每周也能自动汇总，为什么我要用 Logseq 而不用它们？如果你对此感到好奇，可以继续往下看看。

用法四：灵感记录，串联碎片想法

前言，双链的兴起

从 2020 年开始，因为双链这一概念的流行，越来越多的笔记软件开始引入它，其中包括 Notion、字节跳动的幕布和飞书，以及印象笔记都有所沿用。

但我必须说，上述几款的双链都不够好用（有些甚至是不能用），有机会我们再单开一期聊聊这几款。另外，基于双链功能构筑的新笔记软件也开始井喷。既然连大厂都在跟进，我觉得现在你也可以试着体验下，或许就能明白为什么双链这个古老的概念“一夜之间”就火了。

对双链的简单探讨

与双链相对的单向链接，例如在印象笔记中，你虽然可以生成一条笔记的链接并将它嵌入另一条笔记中，使你能够在 B 笔记中看到 A 笔记，但也只能看到标题而已，并且在 A 笔记中，我是看不到 B 的。

所以，Logseq 多了这一层能反向链接到日期的信息，能起到什么用处？

A 和 B 双链，B 和 C 双链，则 A 自动与 C 建立双链，且 B 将作为 A 反链到 C 时的“语境”，这个语境可以是日期、位置、事件、概念，或者是人际关系等。这样一来，你除了能知道 A 和 C 有关系，还知道 A 和 C 是怎么建立关系的。

将这条链路再拉长，A 和 F 乍看之下可能是风马牛不相及的两个概念，但通过中间 B、C、D、E 的链接，你便能补齐从 A 到 F 的所有路径，因为 B、C、D、E 都是现成的“论据”，来自于你过去的每一次不经意的记录，它们共同促成了这一新的发现，给予你新的灵感。

这正式双链带给我的最大价值之一，它给予了我一种信念，让我相信一切记录都有其意义。它不一定适合所有人，但至少它很适合我，这就足够了。

在 daily note 中可以随时随地且不受压力地记录你所认为有价值的信息，不用思考放置文件夹的位置，也不用思考该打什么标签，我只需要将关键词框起来，然后期待碎片的支流终有一天能汇成汪洋大海，我认为这是 Logseq 能让我写下 30 万字的重要原因。

但如果所有东西都放在 daily note 中，真的不会变得很凌乱吗？后面会介绍个让你能有序整理资料的功能。

不受压力地“随手记”

对我来说，有压力的记录大概是这样子的：假设我正在使用印象笔记、或者 bear 或者 notion，突然我有了某种灵感，很多时候我可能不是先把它写下来，而是会先思考它归属于哪个分类，我应该为它建立哪个层级的文件夹？如果它同时涉及多个分类，我是不是应该每个分类都放一份记录？对 MECE （穷尽分类、互相独立）原则的追求，曾一度让我非常痛苦。然而灵感总是稍纵即逝，当我终于把文件夹建好，早已忘记刚才思考的重点。

并且一旦因为某个想法而建立了某个文件夹（或者文档），就会持续地给人一种压力，这种压力类似于“我必须继续往里记东西”。然后未来某天再次看到这个文件夹还是只有零星几条记录时，就会怀疑这个文件夹存在的必要性，甚至嘲笑当初自己信誓旦旦想要大写一场，结果到头来憋不出几条废话，负反馈非常明显，最后干脆归档或者一删了之，眼不见为净。

但上述这些问题，在 Logseq 能够顺其自然地被消解掉，原因就在这一整篇文章当中。

用法五：读书笔记（拆书）

接下来我将以拆书的方式来为这部 166 万字的科幻小说《海伯利安》做一下笔记， 看看在双链和 PDF 标注系统的加持下，我的读书笔记是什么样子的。

Logseq 除了能内嵌视频和图片之外，还支持直接嵌入 PDF 文件，并在左侧边栏中打开进行阅读和标注。我们可以先用书名新建一个 Page，并进入这个 page 当中，然后把这本书的 PDF 文件直接拖曳进去，相当于把这个附件存储在了这个 Page 当中，用过 Notion 的话应该不会对此感到陌生。

我的习惯是，在阅读那些晦涩难懂、或者人物关系错综复杂的书之前，都会先新建三个 page，分别是【故事卡】、【人物卡】、以及【背景卡】，它们的作用分别是：

  • 人物卡：记录所有重要角色
  • 背景卡：记录所有重要设定，包括重要地名、政治派系、科技设定等
  • 故事卡：记录每个重要章节的情节概述

左键【故事卡】，将它在主记录区中打开，shift+鼠标左键，将【背景卡】和【人物卡】打开在右侧边栏中。

这样一来总共就有了从左到右的 3 个工作区：阅读区、主要记录区、侧边记录区。另外，这三个区域的宽度都是可以调整的，个人习惯是给阅读区尽量大的空间。

在阅读的过程中，我决定在【人物卡】中为【领事】和【西奥】这两个人做一些注解，以防我看到后面忘了他们的生平和设定，我们可以继续用 shift+左键点击，将这两个人物的 page 在侧边栏中打开并记录。

你可能注意到，因为我们在【西奥】这个 page 中提到了【领事】，所以两者已经建立了双链关系。所以当进入【领事】这个 page 时，你可以看【西奥】出现在了【领事】的 Linked Reference 区域。

这样一来，不管书中出现的人物再多，你再也不会搞混彼此的关系，他们相互之间都有哪些故事也将一目了然。而且你还可以随时将鼠标悬停在任意一个【关键词】上，它将弹出一个小框，以作提示。

随着阅读进度的推进，我将慢慢补全重要人物，以及所有重要的设定。

你除了可以用自己的语言来概括故事情节，还可以通过选中、并标注 PDF 原文，来将一些不便于概括的设定或者是重要的原文，直接引用（复制）到你的各个 Page 中，它除了会显示所在 PDF 的页数，还可以通过点击引文，直接跳转到 PDF 原来的位置。

我为这部百多万字的小说写了总计近 2.5 万字的情节梗概，尽管过程漫长且低效，但还好并不是每一本书都值得我这么做，这次我再也不会看完一本书却忘了它到底讲了些什么了。

Logseq 中的每一个 Page 都像是《海伯利安》中的一颗星星，通过双链这一功能，它实现了书中【远距传送门】的效果，让我能免去超光速旅行所带来的【时间债】，就像【诗人马丁塞利纳斯】在他的宅邸，他可以清晨在【鲸逖中心】吃早餐，中午一步跨过传送门，就出现在 300 光年外的星光下撰写他那不朽的【诗篇】

我相信这种阅读方法不止适用于读小说，你可以用它来拆解大部头书籍，各种复杂艰深的专有名词也能在你的细致拆解下，融会贯通，了然于心。

结语

关于 Logseq 其实还有很多功能没有介绍到，例如块属性、更详细的 Query 命令、网页嵌入以及油管时间戳等，由于使用门槛较高或者使用频次较低，这里就不多做展示，等到你觉得真的需要这几个功能的那天，可以访问官方说明文档 自行摸索。

其实，面对一款新的笔记软件，我常出现的困扰根本不是“不知道该怎么用”，而是“不知道该写些什么”，或者其实自己根本就没有什么值得记录的东西。就算功能摸索得再熟稔，很多时候它的价值对我来说就只能是产生一种虚无的满足感，让我幻想新的功能可以为我带来新的动力、新的产出。所以曾经我将笔记软件换了一个又一个，正如 GTD 软件，正如记账软件，都是一样的原因。

在这里我并没有打算用 Logseq 来构筑所谓的第二大脑，实际上，每当有人提及“第二大脑”这个词时，我都会怀疑，我这一文不值的脑袋，真能创造出与这个高大上的名词相称的东西吗？当初会用 Logseq 纯粹出于偶然，期望靠它多少获得一种“在记录 ≈ 在进步”的成就感，还好到现在它做得还不错。
